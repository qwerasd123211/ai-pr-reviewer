## PR #2：API 超时机制

**复制以下内容：**

---

## 🎯 功能描述

为 API 调用添加超时机制，防止服务挂起；检测客户端断开连接，避免无效资源消耗。

## 💡 实现思路

### 问题分析
- **服务挂起**：GitHub API 或 Claude API 响应慢时，请求会一直等待
- **资源浪费**：客户端断开后，服务器仍继续处理请求
- **用户体验差**：用户不知道需要等待多久

### 解决方案
1. **超时控制**：使用 `Promise.race` 实现超时包装
2. **断开检测**：监听 `res.close` 事件检测客户端状态
3. **安全写入**：写入前检查连接状态，避免异常

### 超时配置

| API | 超时时间 | 说明 |
|-----|----------|------|
| GitHub API | 30 秒 | 获取 PR 详情和文件变更 |
| Claude API | 120 秒 | AI 分析代码变更 |
| 总超时 | 180 秒 | 整个请求处理时间 |

## 🧪 测试方式

1. **正常流程测试**
   - 输入有效的 PR 链接
   - 验证分析正常完成
   - 检查进度消息是否正确显示

2. **超时测试**
   - 模拟 API 响应慢的场景
   - 验证超时错误是否正确触发
   - 检查错误提示是否友好

3. **断开测试**
   - 在分析过程中关闭浏览器
   - 检查服务器日志是否无异常
   - 验证资源是否正确释放

## 📁 变更文件

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| src/app.js | 修改 | 添加超时控制和断开检测 |

## 🎯 功能描述

支持 Gitee PR 链接解析和分析，同时保留 GitHub 功能，实现多平台代码评审。

## 💡 实现思路

### 需求背景
- 国内开发者常用 Gitee 代码托管平台
- 市面上工具大多只支持 GitHub
- 需要同时支持两个平台，提升适用范围

### 平台识别

```
GitHub: https://github.com/owner/repo/pull/123
Gitee:  https://gitee.com/owner/repo/pulls/123
```

### 数据格式统一

| 字段 | GitHub | Gitee | 说明 |
|------|--------|-------|------|
| title | data.title | data.title | PR 标题 |
| author | data.user.login | data.user.login | 作者 |
| additions | data.additions | parseInt(data.additions) | 新增行数 |
| patch | file.patch | file.patch.diff | 代码差异 |

## 🧪 测试方式

1. **GitHub 测试**
   - 输入：https://github.com/facebook/react/pull/12345
   - 验证：正常获取 PR 信息和代码变更

   - 输入：https://gitee.com/chinabugotech/hutool/pulls/1439
   - 验证：正常获取 PR 信息和代码变更

## 🎯 功能描述

添加 EditorConfig 配置文件，统一代码风格，提升团队协作效率。

## 💡 实现思路

### 什么是 EditorConfig

EditorConfig 是一个用于统一代码风格的配置文件格式，支持多种编辑器。

### 配置内容

```
root = true

[*]
charset = utf-8
end_of_line = lf
insert_final_newline = true
trim_trailing_whitespace = true
indent_style = space
indent_size = 2

[*.md]
trim_trailing_whitespace = false
```

### 优势

1. **统一风格**：所有开发者使用相同代码风格
2. **自动格式化**：编辑器自动应用配置
3. **减少冲突**：减少因格式差异导致的 Git 冲突

## 🧪 测试方式

1. **VS Code 测试**
   - 安装 EditorConfig 插件
   - 打开项目文件
   - 验证缩进是否自动应用

2. **格式化测试**
   - 创建新文件
   - 保存时是否自动格式化
