## 🎯 功能描述

添加 Vercel 部署支持，将项目部署到云端，提供公网访问能力，满足作品提交要求。

## 💡 实现思路

### 需求背景

根据作品提交要求：
> 需自行部署上线并提供可用的后端能力，评委不会自行配置密钥或环境。请确保作品能够直接运行、正常访问，且使用的云端服务已集成完毕，无需额外配置即可体验完整功能。

因此需要将项目部署到公网，让评委可以直接访问和体验。

### 技术选型

选择 Vercel 的原因：
- 免费额度充足
- 自动 HTTPS
- 全球 CDN
- 与 GitHub 无缝集成
- 支持 Serverless 函数

### 架构设计

```
用户访问
    ↓
Vercel CDN（静态文件）
    ↓
┌─────────────────────────────────────┐
│  /api/analyze  →  Serverless 函数   │
│  /index.html   →  静态文件服务      │
└─────────────────────────────────────┘
    ↓
GitHub API / Gitee API / DeepSeek API
```

### 代码改造

问题：原项目使用 Express 服务器，需要长时间运行

解决：改造为 Vercel Serverless 函数

```javascript
// 本地开发时启动服务器
if (process.env.NODE_ENV !== 'production') {
  const PORT = process.env.PORT || 3000;
  app.listen(PORT, () => {
    console.log(`🚀 运行在 http://localhost:${PORT}`);
  });
}

// 导出 app 供 Vercel 使用
module.exports = app;
```

### 配置文件

vercel.json：

```json
{
  "version": 2,
  "builds": [
    {
      "src": "src/app.js",
      "use": "@vercel/node"
    },
    {
      "src": "public/**",
      "use": "@vercel/static"
    }
  ],
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "src/app.js"
    },
    {
      "src": "/(.*)",
      "dest": "public/$1"
    }
  ]
}
```

### 环境变量配置

在 Vercel 控制台配置以下环境变量：

| 变量名 | 说明 | 必需 |
|--------|------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥 | 是 |
| `GITHUB_TOKEN` | GitHub Token（可选） | 否 |
| `GITEE_TOKEN` | Gitee Token（可选） | 否 |
| `NODE_ENV` | 环境标识 | 是 |

## 🧪 测试方式

1. **本地测试**
   ```bash
   npm start
   ```
   访问 http://localhost:3000，验证功能正常

2. **Vercel 部署测试**
   - 推送代码到 GitHub
   - 在 Vercel 中导入项目
   - 配置环境变量
   - 部署并访问

3. **功能验证**
   - 访问 Vercel 分配的域名
   - 输入 PR 链接进行分析
   - 验证所有功能正常工作

4. **性能测试**
   - 测试首次访问延迟（冷启动）
   - 测试并发请求处理
   - 验证 SSE 实时进度推送

## 🚀 部署步骤

1. 合并此 PR
2. 登录 Vercel（https://vercel.com）
3. 导入 GitHub 仓库
4. 配置环境变量
5. 点击部署

## ⚠️ 注意事项

- Vercel Serverless 函数有超时限制（免费版 10 秒）
- 首次访问可能有冷启动延迟（约 1-2 秒）
- 建议配置自定义域名提升访问体验
